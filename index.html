Introduction à la POO en Java

La Programmation Orientée Objet (POO) est un paradigme de programmation qui permet de modéliser des systèmes complexes en les décomposant en entités appelées objets.

Les 4 piliers de la POO :

  1. Encapsulation : Protection des données
  2. Héritage : Réutilisation du code
  3. Polymorphisme : Flexibilité des comportements
  4. Abstraction : Simplification des concepts

Comparaison Programmation Structurée vs POO

Aspect Programmation Structurée POO
Organisation Fonctions et procédures Objets et classes
Données Séparées des traitements Intégrées aux objets
Réutilisation Limitée Élevée (héritage)
Exemple concret d'objet ▼

Considérons un objet CompteBancaire :


// Attributs (état)
private String numero;
private double solde;

// Méthodes (comportement)
public void deposer(double montant) {
    this.solde += montant;
}

public void retirer(double montant) {
    if (montant <= this.solde) {
        this.solde -= montant;
    }
}

Classes et Objets

Une classe est un modèle (blueprint) qui définit la structure et le comportement des objets. Un objet est une instance concrète d'une classe.

Personne
- nom : String
- age : int
+ parler() : void
+ sePresenter() : void

// Définition de la classe
public class Personne {

    // Attributs
    private String nom;
    private int age;

    // Méthodes
    public void parler() {
        System.out.println("Bonjour !");
    }

    public void sePresenter() {
        System.out.println(
            "Je m'appelle " + nom + " et j'ai " + age + " ans.");
    }
}

// Création d'objets
Personne personne1 = new Personne();
personne1.setNom("Alice");
personne1.setAge(25);
personne1.sePresenter();

Bonnes pratiques pour les classes :

  • Nommer les classes avec des noms substantifs (Personne, CompteBancaire)
  • Utiliser PascalCase pour les noms de classes
  • Déclarer les attributs en private
  • Fournir des méthodes publiques pour interagir avec l'objet

Constructeurs

Les constructeurs permettent d'initialiser les objets lors de leur création.


// Définition de la classe
public class Livre {

    // Attributs
    private String titre;
    private String auteur;
    private int annee;

    // Constructeur par défaut
    public Livre() {
        this.titre = "Inconnu";
        this.auteur = "Anonyme";
        this.annee = 0;
    }

    // Constructeur avec paramètres
    public Livre(String titre, String auteur, int annee) {
        this.titre = titre;
        this.auteur = auteur;
        this.annee = annee;
    }

    // Surcharge de constructeur
    public Livre(String titre) {
        this(); // Appelle le constructeur par défaut
        this.titre = titre;
    }
}

Caractéristiques des constructeurs :

  • Même nom que la classe
  • Pas de type de retour (même pas void)
  • Peut être surchargé (plusieurs versions)
  • Peut appeler un autre constructeur avec this()
Exemple complet d'utilisation ▼ 

// Création d'objets avec différents constructeurs
Livre livre1 = new Livre(); // Constructeur par défaut
Livre livre2 = new Livre("Le Petit Prince"); // Constructeur avec titre
Livre livre3 = new Livre("1984", "George Orwell", 1949); // Constructeur complet

System.out.println("Livre 1: " + livre1.getTitre());
System.out.println("Livre 3: " + livre3.getAuteur());

Encapsulation

Principe fondamental qui consiste à protéger les données d'une classe en les rendant privées et en fournissant des méthodes publiques pour y accéder. 


// Définition de la classe
public class CompteBancaire {

    private double solde;

    // Getter
    public double getSolde() {
        return this.solde;
    }

    // Setter avec validation
    public void setSolde(double nouveauSolde) {
        if (nouveauSolde >= 0) {
            this.solde = nouveauSolde;
        } else {
            System.out.println("Erreur: solde négatif");
        }
    }

    // Méthode métier
    public void deposer(double montant) {
        if (montant > 0) {
            this.solde += montant;
        }
    }
}

Pourquoi l'encapsulation ?

  • Contrôle des accès aux données
  • Validation des valeurs
  • Maintenance facilitée
  • Évolution du code sans impact sur les utilisateurs

Héritage

Mécanisme permettant à une classe (classe fille) d'hériter des caractéristiques (attributs et méthodes) d'une autre classe (classe mère). 

                                                                    |--------- Animal ---------|
                                                                    | - nom  : String          |
                                                                    | - age  : int             |
                                                                    |--------------------------|
                                                                    | + manger()   : void      |
                                                                    | + dormir()   : void      |
                                                                    | + parler()   : void      |
                                                                    |--------------------------|
                                                                            ▲
                                                        ┌────────────────────┼────────────────────┐
                                                        │                    │                    │
                                                        ▼                    ▼                    ▼
                                                |------- Chien -------|  |-------- Chat --------|  |------ Oiseau ------|
                                                | - race : String     |  | - couleur : String   |  | - ailes : boolean  |
                                                |---------------------|  |----------------------|  |--------------------|
                                                | + aboyer() : void   |  | + miauler() : void   |  | + chanter() : void |
                                                | + parler() : void   |  | + parler() : void    |  | + parler() : void  |
                                                |---------------------|  |----------------------|  |--------------------|


// Classe mère
public class Animal {

    protected String nom;
    protected int age;

    public void manger() {
        System.out.println(nom + " mange.");
    }
}

// Classe fille
public class Chien extends Animal {

    private String race;

    public void aboyer() {
        System.out.println("Wouf! Wouf!");
    }

    // Redéfinition de méthode
    @Override
    public void manger() {
        super.manger(); // Appel à la méthode parente
        System.out.println("Le chien mange rapidement.");
    }
}

Attention :

  • Java ne supporte pas l'héritage multiple
  • Utilisez super pour accéder à la classe parente
  • Les attributs privés ne sont pas hérités
  • Préférez la composition à l'héritage quand c'est possible

Polymorphisme

Capacité d'un objet à prendre plusieurs formes, permettant d'utiliser une interface unique pour différents types.

Deux types de polymorphisme :

  1. Surcharge (Overloading) : Même nom de méthode, signatures différentes
  2. Redéfinition (Overriding) : Même signature, implémentation différente dans une sous-classe

// Surcharge (Overloading)
public class Calculatrice {
    public int additionner(int a, int b) {
        return a + b;
    }
    
    public double additionner(double a, double b) {
        return a + b;
    }
}

// Redéfinition (Overriding)
public class Animal {
    public void parler() {
        System.out.println("L'animal émet un son.");
    }
}

public class Chien extends Animal {
    @Override
    public void parler() {
        System.out.println("Le chien aboie.");
    }
}

public class Chat extends Animal {
    @Override
    public void parler() {
        System.out.println("Le chat miaule.");
    }
}

// Utilisation du polymorphisme
public class Main {
    public static void main(String[] args) {
        Animal[] animaux = new Animal[3];
        animaux[0] = new Animal();
        animaux[1] = new Chien();
        animaux[2] = new Chat();

        for (Animal animal : animaux) {
            animal.parler(); // Appel de la méthode polymorphe
        }
    }
}

Exercices Pratiques

Exercice 1 : Classe CompteBancaire

Créez une classe CompteBancaire avec :

  • Attributs privés : numeroCompte (String), solde (double), proprietaire (String)
  • Un constructeur initialisant ces attributs
  • Méthodes : deposer(), retirer(), afficherSolde()
  • Ajoutez des validations (solde ne peut pas être négatif)
Correction Exercice 1 ▼ 

// Définition de la classe CompteBancaire
public class CompteBancaire {

    private String numeroCompte;
    private double solde;
    private String proprietaire;

    public CompteBancaire(String numero, String prop) {
        this.numeroCompte = numero;
        this.proprietaire = prop;
        this.solde = 0.0;
    }

    public void deposer(double montant) {
        if (montant > 0) {
            this.solde += montant;
        }
    }

    public boolean retirer(double montant) {
        if (montant > 0 && montant <= this.solde) {
            this.solde -= montant;
            return true;
        }
        return false;
    }

    public void afficherSolde() {
        System.out.println(
            "Solde du compte " + numeroCompte + " : " + solde);
    }
}

QCM d'Évaluation POO Java

Question 1 : Classes et Objets

Quelle affirmation est vraie concernant les classes en Java ?



Réponse correcte : B. Une classe est un modèle (template) pour créer des objets.
Explication : En POO, une classe définit la structure et le comportement que ses instances (objets) auront.

Question 2 : Relation classe-objet

Quelle est la relation entre une classe et un objet ?



Réponse correcte : B. Un objet est une instance d'une classe.
Exemple : Voiture maVoiture = new Voiture(); → maVoiture est un objet instance de la classe Voiture.

Question 3 : Encapsulation

Pourquoi utilise-t-on des getters et setters dans une classe Java ?



Réponse correcte : B. Contrôle d'accès aux attributs.
Avantage : Permet de valider les données avant modification (ex: vérifier qu'un âge n'est pas négatif).

Question 4 : Héritage

Quelle affirmation est fausse concernant l'héritage en Java ?



Réponse correcte : A. Java ne permet pas l'héritage multiple de classes (mais permet d'implémenter plusieurs interfaces).

Question 5 : Polymorphisme

Qu'est-ce que le polymorphisme en Java ?



Réponse correcte : C. Le polymorphisme inclut à la fois la redéfinition de méthodes et la possibilité d'utiliser une référence parente pour un objet enfant.

Question 6 : Constructeurs

Que fait le constructeur par défaut en Java ?



Réponse correcte : B. Le constructeur par défaut appelle implicitement super().

Question 7 : Modificateurs d'accès

Quel modificateur permet d'accéder à un attribut depuis n'importe quelle classe du même package ?



Réponse correcte : A. Le modificateur "package-private" (pas de mot-clé) est le niveau par défaut.

Question 20 : Interfaces

Quelle caractéristique est vraie pour les interfaces Java ?



Réponse correcte : B. Une classe peut implémenter plusieurs interfaces.
Note : Depuis Java 8, les interfaces peuvent avoir des méthodes "default".
